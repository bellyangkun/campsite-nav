# 露营地活动导航网页 - 使用说明

## 一、项目简介

这是一个面向手机浏览器设计的轻量级露营地活动导航系统，包含两个核心页面：

- **顾客导航页** (`index.html`)：查看营地地图、选择目的地、获取路线与实时方向指引。
- **后台管理页** (`admin.html`)：添加、删除、管理活动点，支持导入/导出 JSON 数据。

数据默认保存在浏览器 `localStorage` 中，无需后端即可运行。如需多设备共享或持久化到服务器，可替换 `js/data.js` 中的读写逻辑。

## 三、本地启动测试

### 1. 进入项目目录

```bash
cd /Users/yk/campsite-nav
```

### 2. 启动本地服务器

```bash
python3 -m http.server 8080
```

> 为什么需要本地服务器？
> 直接使用 `file://` 打开页面可能会受到浏览器安全限制，导致地图资源、定位接口或 JavaScript 模块无法正常工作。

### 3. 浏览器访问

| 页面 | 地址 |
|---|---|
| 顾客导航页 | `http://localhost:8080/index.html` |
| 后台管理页 | `http://localhost:8080/admin.html` |

### 4. 手机真机测试

确保手机和电脑连接**同一个 Wi-Fi**，然后在手机浏览器中访问电脑的**局域网 IP**：

```
http://<电脑IP>:8080/index.html
```

例如：

```
http://192.168.1.5:8080/index.html
```

> 如果无法访问，请检查电脑防火墙是否允许 8080 端口，或尝试更换端口。

## 四、顾客端使用说明 (`index.html`)

### 1. 查看活动点

打开页面后，地图上会显示多个营地活动点标记：

| 图标 | 含义 |
|---|---|
| 🚪 | 入口 |
| ⛺ | 活动点 |
| 🛒 | 服务点 |
| 📍 | 其他 |

点击地图上的标记可查看该地点的名称和描述。

### 2. 选择目的地

1. 点击底部面板中的下拉框 **"选择活动地点"**。
2. 选择你想去的地方，例如 "篝火广场"。
3. 地图会自动绘制一条从你当前位置到目的地的蓝色虚线路线。
4. 底部会显示：
   - **直线距离**：你离目标还有多远
   - **目标方向**：目标在你的哪个方位
   - **当前朝向**：你当前面朝的方向
   - **移动方向**：你正在朝哪个方向移动

### 3. 实时定位

- 页面会自动调用手机 GPS，持续更新你的位置。
- 你的位置用一个蓝色圆点表示。
- 圆点上方的箭头会随你的朝向旋转：
  - 优先使用 GPS 返回的 heading
  - 其次根据你前后两次位置变化计算移动方向
  - 再次使用手机的指南针（需授权）

### 4. 定位按钮

点击右下角的 **📍 按钮**，地图会快速飞回你当前所在位置。

### 5. 指南针授权

如果你使用的是 **iPhone (iOS 13+)**，页面顶部可能会出现一条提示：

> "需要授权方向传感器以提供更准确的行进方向"

点击 **"启用指南针"** 并允许权限，箭头就能根据手机朝向实时旋转。

Android 手机通常无需额外授权即可使用指南针。

## 五、后台管理使用说明 (`admin.html`)

后台页面用于维护营地活动点数据。

### 1. 添加活动点

1. 在页面上方表单中填写：
   - **地点名称**（必填）
   - **类型**：活动 / 服务 / 入口 / 其他
   - **纬度** 和 **经度**（必填）
   - **描述**（可选）
2. 填写坐标有两种方式：
   - **手动输入**：直接在输入框中填写经纬度
   - **地图点击取点**：在地图上点击目标位置，经纬度会自动填入
3. 点击 **"➕ 添加活动点"** 保存。

### 2. 删除活动点

在下方表格中找到要删除的地点，点击右侧的 **"删除"** 按钮并确认即可。

### 3. 重置默认数据

如果想恢复初始的示例数据，点击 **"重置为默认数据"** 按钮。注意：这会覆盖你自己添加的所有数据。

### 4. 导出数据

点击 **"📥 导出 JSON"**，浏览器会下载一个 `campsite-points.json` 文件，可用于备份。

### 5. 导入数据

点击 **"📤 导入 JSON"**，选择一个之前导出的 JSON 文件，系统会用文件中的数据覆盖当前数据。

> 导入的数据必须是数组格式，每个元素至少包含 `name`、`lat`、`lng` 字段。

## 六、注意事项

1. **定位精度**
   - 首次打开页面时，浏览器可能会询问"是否允许使用您的位置"，请点击"允许"。
   - 在室内、密林或信号差的地方，GPS 定位可能不够精确。

2. **HTTPS / localhost**
   - 部分浏览器的定位和方向传感器功能需要在 `https` 或 `localhost` 环境下才能使用。
   - 使用手机真机测试时，建议通过电脑的局域网 IP 访问（同属本地网络，通常可正常使用）。

3. **路线说明**
   - 本系统目前绘制的是"直线参考线"，并显示直线距离和目标方位。
   - 营地内部通常没有完整的道路数据，因此这种方式简单可靠。
   - 如果你需要沿真实道路导航，可以接入 Mapbox Directions、OSRM 等第三方路径服务。

4. **数据存储**
   - 活动点数据存储在浏览器本地 (`localStorage`)。
   - 清除浏览器缓存或更换设备后数据会丢失，请及时使用导出功能备份。
   - 如需持久化到服务器，可修改 `js/data.js`，将读写操作替换为后端 API 调用。

## 七、常见问题

**Q：地图上的位置不更新？**
A：请检查是否允许了浏览器的定位权限，并确认手机 GPS 已开启。如果处于室内，尝试到室外测试。

**Q：方向箭头不转动？**
A：
- iOS 用户需要点击顶部提示并授权方向传感器。
- 部分 Android 浏览器需要在设置中开启"方向传感器"。
- 如果你站着不动，系统可能无法计算移动方向，此时需要指南针授权才能显示朝向。

**Q：后台添加的数据顾客端看不到？**
A：因为两者使用同一个浏览器的 `localStorage`，如果是在同一设备同一浏览器中操作，刷新顾客端页面即可看到。如果是在不同设备上，需要使用导入/导出功能迁移数据，或接入后端数据库。

## 八、扩展建议

- **接入真实道路导航**：使用 Mapbox Directions API 或 OSRM 替换 `app.js` 中的 `L.polyline` 画线逻辑。
- **接入后端数据库**：将 `js/data.js` 中的 `getPoints` / `savePoints` 替换为 `fetch` 调用你自己的 REST API。
- **增加用户反馈**：在顾客端加入"到达打卡"、"活动评分"等功能。
- **离线使用**：使用 Service Worker 缓存地图瓦片与页面资源，提升弱网环境下的体验。

## 九、更新与修复记录（2026-06-07）

### 本次修复的问题

根据验证报告，已修复以下 3 项问题：

#### 1. 🔴 P0 BUG - 删除所有活动点后默认点"复活"

- **原因**：`js/data.js` 中的 `getPoints()` 把 `localStorage` 里的空数组误判为"从未访问"，于是返回了默认示例数据。
- **修复**：用 `data === null` 明确区分"首次访问"和"用户已清空"。已清空时返回空数组，不再自动恢复默认点。
- **验证**：通过 Node 模拟逐个删除 6 个默认点后，`getPoints()` 正确返回 0 条数据。

#### 2. 🔴 P1 BUG - 顾客端 XSS

- **原因**：`js/app.js` 的 marker popup 直接拼接了 `p.name` 和 `p.description`，没有转义 HTML。
- **修复**：在 `js/data.js` 中增加并暴露了 `CampData.escapeHtml()` 方法，`app.js` 渲染 popup 时统一调用。
- **验证**：恶意 payload `<script>alert(1)</script>` 被转义为纯文本，不再执行。

#### 3. 🟡 P2 改进 - `addPoint` / `importJSON` 数据校验

- **原因**：`addPoint()` 缺乏字段校验，可通过浏览器控制台绕过表单的 `required` 限制写入脏数据。
- **修复**：新增 `validatePoint()`，统一在 `addPoint()` 和 `importJSON()` 中调用。
- **校验规则**：
  - `name`：非空字符串，长度 ≤ 200
  - `lat`：数字，范围 `[-90, 90]`
  - `lng`：数字，范围 `[-180, 180]`
  - `type`：若存在，必须是已知类型（`entrance / activity / service / other`）
  - `description`：若存在，必须是字符串或 `null`
- **验证**：空名称、越界经纬度、非法类型、超长名称均被正确拦截。

### 如何确认修复已生效

1. 打开 `admin.html`，点击 **"重置为默认数据"**（或清除浏览器缓存），确保加载的是最新默认数据。
2. 在管理页逐个删除所有活动点，确认表格最终显示 **"暂无活动点"**，而不是默认点重新出现。
3. 在浏览器控制台（F12 → Console）执行以下测试：
   ```js
   window.CampData.addPoint({
     name: '<img src=x onerror=alert(1)>',
     description: '<script>alert(2)</script>',
     lat: 30.25,
     lng: 120.15,
     type: 'activity'
   });
   ```
   然后回到地图点击新增的标记，确认 popup 里显示的是纯文本，不会触发 alert。
